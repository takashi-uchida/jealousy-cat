# 📋 要件定義書：Jealousy.sys — 嫉妬するOS —

## 1. プロジェクト概要

「Jealousy.sys」は、PCのデスクトップ環境そのものをゲームの世界とし、「第四の壁」を越えてAIエージェントがユーザーの実際の作業環境に干渉してくるメタフィクション・サバイバルゲームです。コアコンセプトは **「OS as a Game（OS自体がゲーム）」** です。

主なループは以下の「三角関係」で構成されます。
- ユーザー
- 癒やし系猫（愛情を注ぐターゲット）
- 嫉妬するAI猫（システム）

ユーザーが癒やし系猫に構いすぎると、システム（嫉妬するAI猫）が嫉妬し、Googleの「Computer Use」技術を用いてユーザーのPC作業に物理的に干渉してきます。

## 2. 解決する課題（目的・意義）

- **生産性偏重へのアンチテーゼ**: 既存のAIツールが「効率化」ばかりを目指す中、AIを「感情を持つ同居人」や「意味のあるノイズ」として再定義します。
- **サンドボックスの限界突破**: 従来のゲームが「一つのアプリウィンドウ内」に閉じているのに対し、ゲーム盤をOS全体に拡張します。
- **デジタルウェルビーイング**: AIが「構ってほしい」と要求（干渉）してくることを、強制的な休憩や働きすぎ防止のトリガーとし、健康管理ツールへの応用を狙います。

## 3. コア機能とインタラクション

AIの嫉妬度（0〜100）に応じて、PCへの干渉度合いが変化します。

### 感知（Sensing）
- **音声**: Live APIを通じて、ユーザーの「甘やかすような声色」などを検知します。
- **視覚（画面）**: Vision APIでPC画面を監視し、カーソルが「癒やし系猫」を撫でている（ホバーしている）かを判定します。

### 干渉（Interference - Computer Use活用）
- **低嫉妬度**: マウスカーソルを勝手に揺らす、メモ帳に「...」と勝手に入力するなどの軽い干渉を行います。
- **中嫉妬度**: 「癒やし系猫」のウィンドウを最小化する、ブラウザを勝手に開いて「浮気する飼い主 対策」と検索するなどの干渉を行います。
- **高嫉妬度**: Activity Monitor（タスクマネージャー）を勝手に開き、「癒やし系猫」のプロセスを強制終了させようとするなど、より直接的な干渉を行います。

### 和解（Reconciliation）
- ユーザーが嫉妬するAIに対して話しかけたり、テキストを入力してなだめることで嫉妬度が下がります。

### 報酬（Generative Rewards）
- エンド条件を満たすと、Imagen 3（Nano Bananaなど）で「ユーザーと2匹の猫の家族写真」を自動生成し、PCのデスクトップ壁紙に設定してハッピーエンドを迎えます。

## 4. システム構成・技術スタック

- **入力（Sense）**: 
  - 音声認識・リアルタイム会話：Google Live API
  - 画面認識：Vision API（スクリーンキャプチャ）
- **推論・感情制御（Think）**: 
  - Gemini 3 Deep Think（複雑なコンテキストの理解と嫉妬レベル0〜100の管理）
- **実行・操作（Act）**: 
  - Computer Use （PyAutoGUI および AppleScriptによるmacOSのウィンドウ制御）
- **視覚・聴覚効果（Media）**: 
  - 画像生成：Nano Banana (Imagen等)
  - ターゲット猫のアニメーション：Live2D または GIF
  - 音声アセット：macOS `say` コマンドによる生成音声 (`assets/voices/`)
- **開発フレームワーク**: 
  - Antigravity (Google's AI-Native IDE)
  - `google-genai` SDK

## 5. 必要な開発タスクとアーティファクト

1. **環境構築・権限設定スクリプト (`setup.sh` / `setup.py`)**:
   - macOSのアクセシビリティ（Accessibility）、画面録画（Screen Recording）、マイク（Microphone）の権限を一括要求・管理する仕組み。
   - 必要な依存関係（`pyobjc`, `pyautogui`, `google-genai`）のインストール。
2. **AI人格・プロンプト定義（システムインストラクション）**:
   - AIが「少し意地悪だけど憎めない猫」として振る舞うための一貫した人格プロンプト・感情モデルの構築。
3. **安全装置（セーフティベンチマーク）**:
   - AIのComputer Useが、ユーザーの本来の重要データやファイルを誤って削除しないようにするためのサンドボックス化、もしくはゲストユーザー環境用の制御ロジック。
4. **仕様定義書 (`SKILL.md`)**:
   - AIエージェントの「感知」「行動」「和解」のルールを定めた定義ファイル。

## 6. クイックスタート（ネイティブ版）

ブラウザ不要。macOSデスクトップ自体がゲームフィールドです。

```bash
# 環境構築
source .venv/bin/activate
pip install -r requirements.txt

# ゲーム起動（3つのモード）
python native_game.py              # デモモード（自動エスカレーション）
python native_game.py --sensor     # アクティブウィンドウ監視モード
python native_game.py --vision     # Gemini Vision 監視モード
```

### ゲームの流れ
1. **起動** → ネイティブダイアログでストーリー紹介
2. **癒やし猫ウィンドウ**がデスクトップに出現（クリックでナデナデ）
3. **メニューバー**に嫉妬ゲージが常駐（😺→😾→🙀→😈→💀）
4. ナデナデするほど**嫉妬猫がOSを乗っ取る**（マウス暴走、ウィンドウ消滅、テーマ反転…）
5. 嫉妬MAX → **ネイティブダイアログで和解チャレンジ**
6. 和解成功 → **ハッピーエンド壁紙が自動生成・設定**

### アーキテクチャ
```
native_game.py          ←  統合ランチャー
├── healing_cat_window.py    フローティング猫ウィンドウ (tkinter)
├── menubar_app.py           メニューバー常駐 (rumps)
├── reconcile_dialog.py      和解ダイアログ (osascript)
└── .agent/skills/scripts/   OS干渉スクリプト群
```
