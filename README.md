# 📋 Requirement Specification: Jealousy.sys — The Jealous OS —

## 1. Project Overview

"Jealousy.sys" is a metafiction survival game that turns the PC desktop environment itself into a game world. It breaks the "fourth wall" as an AI agent interferes with the user's actual work environment. The core concept is **"OS as a Game"**.

The main loop consists of the following "love triangle":
- The User
- The Healing Cat (Target of affection)
- The Jealous AI Cat (The System)

If the user gives too much attention to the Healing Cat, the system (Jealous AI Cat) becomes jealous and physically interferes with the user's PC work using Google's "Computer Use" technology.

## 2. Problem to Solve (Purpose/Significance)

- **Antithesis to Productivity Focus**: While existing AI tools aim solely for "efficiency," this project redefines AI as an "emotional housemate" or "meaningful noise."
- **Breaking Sandbox Limits**: Unlike traditional games confined within a "single app window," the game board is extended to the entire OS.
- **Digital Well-being**: The AI's demand for attention (interference) acts as a trigger for forced breaks or prevention of overworking, aiming for applications in health management tools.

## 3. Core Features and Interaction

The intensity of PC interference changes according to the AI's jealousy level (0-100).

### Sensing
- **Audio**: Detects "pampering tones" from the user via the Live API.
- **Vision (Screen)**: Monitors the PC screen via the Vision API to determine if the cursor is petting (hovering over) the "Healing Cat."

### Interference (Using Computer Use)
- **Low Jealousy**: Minor interferences like shaking the mouse cursor or typing "..." into Notepad.
- **Medium Jealousy**: Interferences like minimizing the "Healing Cat" window or opening a browser to search for "how to deal with a cheating owner."
- **High Jealousy**: More direct interferences like opening the Activity Monitor and attempting to force-quit the "Healing Cat" process.

### Reconciliation
- The jealousy level decreases when the user talks to or types soothing messages to the jealous AI.

### Generative Rewards
- Upon reaching the end condition, "family photos of the user and the two cats" are automatically generated using Imagen 3 (Nano Banana, etc.) and set as the PC desktop wallpaper for a happy ending.

## 4. System Configuration / Technology Stack

- **Input (Sense)**: 
  - Voice Recognition & Real-time Conversation: Google Live API
  - Screen Recognition: Vision API (Screen Capture)
- **Reasoning & Emotion Control (Think)**: 
  - Gemini Deep Think (Understanding complex contexts and managing jealousy levels 0-100)
- **Execution & Operation (Act)**: 
  - Computer Use (macOS window control via PyAutoGUI and AppleScript)
- **Visual & Audio Effects (Media)**: 
  - Image Generation: Nano (Imagen, etc.)
  - Target Cat Animation: GIF or Sprite animation
  - Audio Assets: Voice generated by macOS `say` command (`assets/voices/`)
- **Development Framework**: 
  - Antigravity (Google's AI-Native IDE)
  - `google-genai` SDK

## 5. Required Development Tasks and Artifacts

1. **Environment Setup & Permission Scripts (`setup_env.sh` / `permissions_check.py`)**:
   - Mechanism to request and manage macOS Accessibility, Screen Recording, and Microphone permissions.
   - Installation of necessary dependencies (`pyobjc`, `pyautogui`, `google-genai`).
2. **AI Personality & Prompt Definition (System Instruction)**:
   - Construction of a consistent personality prompt and emotional model for the AI to behave as a "slightly mischievous but lovable cat."
3. **Safety Mechanism (Safety Benchmarks)**:
   - Sandboxing or guest user environment control logic to ensure the AI's Computer Use doesn't accidentally delete important user data or files.
4. **Specification Document (`SKILL.md`)**:
   - Definition file establishing the rules for the AI agent's "Sensing," "Action," and "Reconciliation."

## 6. Quick Start (Native Version)

No browser required. The macOS desktop itself is the game field.

```bash
# Environment Setup
source .venv/bin/activate
pip install -r requirements.txt

# Launch Game (3 Modes)
python native_game.py              # Demo Mode (Auto-escalation)
python native_game.py --sensor     # Active Window Monitoring Mode
python native_game.py --vision     # Gemini Vision Monitoring Mode
```

### Game Flow
1. **Launch** → Story introduction via native dialog.
2. **Healing Cat Window** appears on the desktop (Click/Hover to pet).
3. **Jealousy Gauge** resides in the menu bar (😺→😾→🙀→😈→💀).
4. The more you pet, the more the **Jealous Cat takes over the OS** (mouse frenzy, window disappearance, theme inversion...).
5. Max Jealousy → **Reconciliation Challenge** via native dialog.
6. Reconciliation Success → **Happy Ending Wallpaper** is automatically generated and set.

### Architecture
```
native_game.py          ← Integrated Launcher
├── healing_cat_window.py   Floating Cat Window (tkinter)
├── menubar_app.py          Menu Bar App (rumps)
├── reconcile_dialog.py     Reconciliation Dialog (osascript)
└── .agent/skills/scripts/  OS Interference Scripts
```
